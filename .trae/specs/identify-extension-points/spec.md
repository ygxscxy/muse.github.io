# 功能扩展点与修复点 Spec

## Why
对 Muse Markdown 桌面应用进行全面代码审查，识别可改进的功能扩展点和需要修复的问题点，提升代码质量和用户体验。

## What Changes
- 识别并分类项目中的功能扩展机会和代码问题
- 按优先级排序，提供可执行的改进建议
- 涵盖代码质量、功能缺失、用户体验、架构优化等维度

## Impact
- Affected specs: 代码质量、编辑器功能、用户体验、系统稳定性
- Affected code: 全局性影响，涉及主进程、渲染进程、IPC通信、编辑器扩展等

## 分析结果

### 一、代码质量问题（需修复）

#### 1. ESLint 错误（高优先级）
当前项目存在 **12 个 ESLint 错误** 和 **13 个警告**，需要修复：

**路径解析错误**（5处）：
- `src/main/index.ts` 导入 `.js` 扩展名但实际文件为 `.ts`
- `src/main/ipc/app.ts`、`dialog.ts`、`file.ts`、`preload.ts` 等存在同样问题
- **影响**：ESLint 报错，但构建可能正常（Vite 处理）

**未使用的变量/导入**（11处）：
- `src/renderer/components/Editor/extensions/codeBlock.tsx`：`hljs` 被导入但未使用（9处）
- `src/renderer/components/Toolbar/index.tsx`：`onSaveAs` prop 未使用
- `src/preload/index.ts`：使用了 `any` 类型

**代码风格问题**：
- `src/main/menu/index.ts:159`：不必要的分号
- `src/renderer/utils/helpers.ts:10`：不必要的类型注解

#### 2. 潜在 Bug

**路径处理硬编码问题**：
```typescript
// src/renderer/App.tsx:138
const parentPath = filePath.split(/[\\/]/).slice(0, -1).join("\\")
// 使用 Windows 风格的反斜杠，在 macOS/Linux 上可能出错
```
类似代码出现在多处（App.tsx、Sidebar/index.tsx）。

**Mermaid 图表渲染使用随机 ID**：
```typescript
// src/renderer/components/Editor/extensions/mermaid.tsx:31
const id = `mermaid-${Math.random().toString(36).slice(2)}`
```
每次重新渲染都会生成新 ID，可能导致 DOM 泄漏和性能问题。

### 二、功能缺失（可扩展）

#### 1. 自动保存功能（中高优先级）
**现状**：设置面板显示"自动保存"选项，但标记为"开发中"，未实现。
**影响**：用户可能因意外关闭而丢失编辑内容。
**建议**：实现基于防抖的自动保存机制，支持配置保存间隔。

#### 2. 全局搜索/查找替换（中优先级）
**现状**：编辑器缺少查找替换功能（Cmd/Ctrl+F）。
**影响**：在长文档中定位内容困难。
**建议**：集成 TipTap 的查找替换插件或自定义实现。

#### 3. 导出功能（中优先级）
**现状**：缺少导出为 PDF/HTML 的功能。
**影响**：用户无法将文档分享给非 Markdown 用户。
**建议**：使用 Electron 的 webContents.printToPDF() 实现 PDF 导出。

#### 4. 最近打开文件列表（低优先级）
**现状**：无最近文件记录。
**影响**：用户无法快速重新打开之前编辑的文档。
**建议**：使用 electron-store 保存最近打开的 5-10 个文件路径。

#### 5. 多标签页支持（低优先级）
**现状**：一次只能编辑一个文件。
**影响**：需要在多个文档间切换时效率低。
**建议**：实现标签页系统，支持多文件同时编辑。

#### 6. 图片上传/本地图片支持（中优先级）
**现状**：图片仅支持 URL 或 base64 嵌入。
**影响**：本地图片无法直接引用，base64 会导致文件体积膨胀。
**建议**：支持将图片复制到项目目录并生成相对路径引用。

### 三、用户体验改进

#### 1. 缺少用户反馈
**问题**：文件操作（保存、删除、重命名）失败时只在 console 打印错误，用户看不到提示。
**建议**：使用 Electron 的 dialog.showErrorBox 或自定义 toast 通知。

#### 2. 未保存更改警告
**问题**：关闭有未保存更改的文件时没有提示。
**建议**：在关闭窗口前检查 isDirty 状态并显示确认对话框。

#### 3. 状态栏信息不足
**现状**：状态栏显示文件名、字数、字符数、光标位置。
**可扩展**：添加阅读时间估计、Markdown 元素统计（标题数、图片数等）。

### 四、架构优化

#### 1. IPC 通信模式
**现状**：使用 ipcMain.handle + ipcRenderer.invoke 模式，良好。
**可扩展**：考虑添加请求验证和类型安全的 IPC 层。

#### 2. 状态管理
**现状**：使用 Zustand，结构清晰。
**可扩展**：考虑添加状态持久化中间件，减少手动保存逻辑。

#### 3. 文件监听性能
**现状**：watch 事件触发时重新构建整个文件树（refreshTree）。
**问题**：大文件夹下性能可能较差。
**建议**：增量更新文件树而非全量重建。

## 优先级总结

### P0 - 必须修复
1. ESLint 错误修复
2. 路径处理跨平台兼容性
3. 未保存更改关闭警告

### P1 - 强烈建议
1. 自动保存功能实现
2. 用户操作反馈（错误提示）
3. Mermaid 渲染 ID 问题修复

### P2 - 推荐扩展
1. 查找替换功能
2. PDF/HTML 导出
3. 本地图片支持
4. 文件监听性能优化

### P3 - 锦上添花
1. 最近打开文件列表
2. 多标签页支持
3. 状态栏增强
