# Muse 落地页 v1.1.0 扩展 Spec

## Why

Muse 编辑器已发布至 v1.0.1，落地页当前版本停留在 v1.0.0 阶段。需要为即将发布的 v1.1.0 版本进行内容丰富和视觉升级，增强产品说服力和品牌质感，同时补充缺失的页面（如更新日志条目、路线图等），使宣传站点更完整、更有层次。

## Visual Thesis（v1.1.0 更新）

在现有干净专业的基础上增加**叙事深度**：用真实截图替代纯 Mock 编辑器、用数据条建立信任感、用版本亮点区展示迭代活力。整体保持深色基调 + Inter 字体 + 蓝色强调色不变，新增紫-蓝渐变用于新功能高亮区域。

## Content Plan（v1.1.0 更新后完整结构）

1. **Hero** — 品牌名 Muse + 标题 + 副标题 + 下载 CTA（更新版本号至 v1.1.0）
2. **Features** — 三大核心能力卡片（保持不变）
3. **Highlights** — 六项快捷特性网格（保持不变）
4. **🆕 What's New** — v1.1.0 版本亮点展示区（全新），3 个关键更新带图标说明
5. **🆕 Stats Strip** — 数据信任条：下载量 / GitHub Stars / 支持平台数 / 文件格式数（全新）
6. **Demo** — 编辑器界面展示（增强：叠加真实截图或改进 Mock 细节）
7. **🆕 Tech Stack** — 技术栈展示条（全新）：Electron · React · TipTailwindCSS · VitePress
8. **CTA** — 下载引导（更新版本号至 v1.1.0）
9. **Footer** — 版权信息

## Interaction Thesis（v1.1.0 新增）

1. Hero 区域截图的视差滚动效果（scroll-linked translateY）
2. Stats Strip 数字从 0 到目标值的滚动计数动画（IntersectionObserver 触发）
3. What's New 卡片交错入场动画（staggered fade-in）

## What Changes

### 页面级变更
- `LandingLayout.vue` — 新增 What's New、Stats Strip、Tech Stack 三个 Section；Hero 区域增加真实截图；所有 v1.0.0 引用更新为 v1.1.0
- `style.css` — 新增三个新 Section 的样式变量与组件样式；新增数字计数动画；新增视差滚动样式
- `custom.css` — 同步清理旧版残留样式（如有冲突）

### 新增页面
- `changelog/v1.1.0.md` — v1.1.0 版本更新日志
- `changelog/index.md` — 更新版本列表加入 v1.1.0
- `guide/roadmap.md` — 产品路线图页面（可选）

### 配置变更
- `config.mts` — 导航栏可能新增「路线图」入口；meta 信息更新

### 素材
- 新增/替换截图素材以匹配 v1.1.0 界面

## ADDED Requirements

### Requirement: v1.1.0 What's New 区域

The system SHALL 在 Highlights 和 Demo 之间插入一个「What's New in v1.1.0」区域。

#### Scenario: 版本亮点展示
- **WHEN** 用户滚动到该区域
- **THEN** 展示 3 个 v1.1.0 关键更新项，每项包含：
  1. 图标 + 标题「插件化扩展系统」— 描述支持第三方扩展的能力
  2. 图标 + 标题「云端同步预览」— 描述多设备数据同步的前瞻能力
  3. 图标 + 标题「性能大幅提升」— 描述启动速度、编辑响应等性能优化
- 卡片采用左图标右文字的水平布局，带边框和 hover 效果
- 卡片之间有交错延迟的淡入动画

### Requirement: Stats Strip 数据信任条

The system SHALL 在 What's New 和 Demo 之间插入一个全宽数据统计条。

#### Scenario: 数据展示
- **WHEN** 用户滚动到该区域
- **THEN** 以水平排列的方式展示 4 组数据：
  1. 「10,000+」下载次数
  2. 「500+」GitHub Stars
  3. 「3」大平台支持
  4. 「190+」代码高亮语言
- 每个数字在进入视口时从 0 滚动到目标值（Counter Animation）
- 数据下方有简短标签说明
- 深色背景区分于上下文

### Requirement: Tech Stack 技术栈展示

The system SHALL 在 Demo 和 CTA 之间插入技术栈展示条。

#### Scenario: 技术栈展示
- **WHEN** 用户查看该区域
- **THEN** 以简洁的文字链形式展示：「Built with Electron · React · TipTap · TailwindCSS · VitePress」
- 技术名称使用等宽字体，带有微妙的背景色区分
- 整体居中排列，不抢夺视觉焦点

### Requirement: Hero 区域真实截图

The system SHALL 在 Hero 内容下方添加编辑器真实界面截图。

#### Scenario: 截图展示
- **WHEN** 用户访问首页
- **THEN** Hero 区域在 CTA 按钮下方显示一张圆角编辑器完整界面截图
- 截图带有微妙的上浮 hover 效果和阴影变化
- 截图随滚动产生轻微的视差位移（parallax effect）

### Requirement: 全局版本号更新

The system SHALL 将所有文件中的版本号引用从 v1.0.0/v1.0.1 更新为 v1.1.0。

#### Scenario: 版本一致性
- **WHEN** 用户查看任何页面
- **THEN** 所有可见的版本号统一显示为 v1.1.0：
  - LandingLayout.vue 中 hero-version 和 cta-version
  - download.md 中的版本信息和日期
  - changelog 新增 v1.1.0 条目

### Requirement: v1.1.0 更新日志

The system SHALL 创建 v1.1.0 版本的更新日志文件。

#### Scenario: 更新日志内容
- **WHEN** 用户访问更新日志页面
- **THEN** 可看到 v1.1.0 条目，包含：
  - 发布日期
  - 新增功能列表（不少于 5 项）
  - 优化与修复列表
  - 各平台下载链接

## MODIFIED Requirements

### Requirement: LandingLayout.vue 结构扩展

原 LandingLayout 包含 Hero → Features → Highlights → Demo → CTA → Footer 共 6 个区块。

**修改为**：Hero → Features → Highlights → **What's New** → **Stats Strip** → Demo → **Tech Stack** → CTA → Footer 共 9 个区块。

### Requirement: style.css 样式扩展

原 style.css 定义了 light/dark 两套主题变量和各区块样式。

**新增**：
- `.whats-new-section` 及其子元素样式（卡片布局、交错动画）
- `.stats-strip-section` 及其子元素样式（数字计数器、全宽深色背景）
- `.tech-stack-section` 样式（文字链布局）
- `.hero-screenshot` 样式（视差滚动容器）
- Counter 动画 keyframes
- Parallax scroll 相关 JS 逻辑所需的 CSS 变量
